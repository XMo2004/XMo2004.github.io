# 全站分类与标签胶囊紧凑化设计

日期：2026-07-15
状态：已获用户确认，待实现

## 1. 目标

将全站分类、专栏和标签胶囊从当前 44px 视觉高度统一缩短为 36px，使文章头部和标签列表更紧凑，同时建立可复用的共享规范，确保后续新增同类组件继续使用相同高度。

## 2. 适用范围

本次统一调整以下组件：

- 文章头部的分类胶囊，例如“成长”。
- 文章头部的专栏胶囊。
- `TagList` 在文章页及其他页面输出的普通、紧凑标签胶囊。

搜索按钮、主题切换按钮、对话框关闭按钮和主导航等操作控件不属于分类或标签胶囊，不随本次调整缩短。

## 3. 视觉规范

- 胶囊可见高度统一为 36px（`2.25rem`）。
- 保留现有字体大小、字重、颜色、边框和全圆角造型。
- 保留现有横向内边距，避免文字与边框过近。
- 胶囊之间继续使用现有间距，不改变换行行为。
- 分类实底、专栏线框和标签柔和线框的语义层级保持不变。

## 4. 交互与可访问性

可见胶囊缩短后，通过透明的扩展命中区域维持约 44px 的垂直操作范围。扩展区域不得改变可见高度、正常文档流或胶囊之间的视觉间距，也不得遮挡相邻链接。

键盘焦点、悬停和激活状态继续作用于完整链接；现有文字对比度与语义化链接结构不变。

## 5. 实现结构

新增一套共享的分类标签胶囊约束：

- 在全局设计令牌中定义 36px 胶囊高度。
- 为分类、专栏和标签链接使用统一的基础胶囊类。
- 各变体只负责配色、字重、字号和横向间距等差异，不再分别声明高度。
- 后续新增分类或标签类胶囊时复用基础类和高度令牌，避免再次出现局部 44px 高度。

现有操作控件继续使用独立的 44px 控件尺寸，不与分类标签胶囊令牌混用。

## 6. 测试与验收

- 源码测试验证共享胶囊类与 36px 高度令牌存在。
- 源码测试验证文章分类、专栏和 `TagList` 均复用共享胶囊规范。
- 源码测试防止这些胶囊重新声明为 44px 高度。
- 运行完整测试、Astro 类型检查和生产构建。
- 在文章页和至少一个标签聚合页面检查桌面与手机布局。
- 确认胶囊可见高度为 36px、文字垂直居中、换行正常、焦点清晰且控制台无新增错误。

## 7. 非目标

- 不改变内容数据、路由或 Markdown 结构。
- 不重新设计胶囊颜色、字体或交互动画。
- 不压缩导航、搜索、主题切换及其他非分类标签控件。
- 不发布或部署线上站点；部署仍沿用项目现有流程。
