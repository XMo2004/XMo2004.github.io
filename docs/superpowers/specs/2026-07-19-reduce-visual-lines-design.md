# 全站柔和纸面减线视觉优化设计

日期：2026-07-19
状态：已获用户确认，待实现

## 1. 目标

在不改变博客信息架构、内容模型和交互逻辑的前提下，减少全站重复分隔线、卡片描边和内部边界，以“柔和纸面分区”重新组织视觉层级，使页面更安静、完整，同时保留现有暖纸色、宋体、砖红、苔绿和编辑式排版气质。

本轮成功标准：

- 同一层级只使用一种主要分隔手段，优先顺序为留白、排版、低对比底色、必要边线。
- 一级内容组使用柔和纸面色差；组内列表和链接默认无框，仅在悬停、焦点或选中时增强反馈。
- 不把删除的边线普遍替换成阴影或更多小卡片。
- 键盘可用性、触控面积、正文可读性、飞书内容语义和明暗主题对比度不退化。

## 2. 现状与问题

当前样式中约有 38 处以 `--line` 绘制的可见边界，其中大部分集中在 `src/styles/global.css`。杂乱感主要来自多层边界叠加，而不是单条线颜色过深：

- 首页同时使用首屏底线、分类区底线、栏目标题底线、列表顶线和列表项底线。
- 文章列表同时使用列表顶线、逐项底线和悬停左侧强调线。
- 精选卡片同时拥有外框、阴影和封面内部隔线。
- 分类、专栏和标签目录为每个项目绘制完整外框。
- 文章页同时使用头部底线、目录顶线、文章导航顶线与卡片框、相关推荐顶线与逐项底线。
- 搜索弹窗同时使用外框、标题底线、输入框、结果分隔线和当前项轮廓。

因此不能简单整体调淡 `--line`。这会同时削弱输入框、表格、引用、焦点和飞书语义块等必要边界，却不能解决重复层级问题。

## 3. 已确认方向

用户在轻度、平衡、极简三档中选择“平衡”，并在三种平衡方案中选择“柔和纸面分区”。

设计原则：

- 一级区块获得轻微纸面色差和统一圆角。
- 二级项目依靠排版、间距与交互底色，不使用常驻描边。
- 精选内容和弹窗允许一层轻阴影；普通区块不使用阴影。
- 正文阅读区域保持原始纸张背景，不放进大卡片。
- 明暗主题使用各自明确的纸面 token，不通过降低透明度形成不可控叠色。

## 4. 视觉系统

### 4.1 色彩 token

保留现有 `--paper`、`--paper-raised`、`--ink`、`--muted`、`--line`、`--terracotta` 和 `--sage`。新增两类专用 token：

- `--paper-soft`：一级内容组的柔和纸面。
  - 浅色主题：`#f8f4ec`。
  - 深色主题：`#1c201c`。
- `--paper-interactive`：无框项目悬停、选中和内层纸面的背景。
  - 浅色主题：`#eee8dd`。
  - 深色主题：`#252a25`。

`--line` 的值保持不变，只继续服务必要边界。新增 token 必须纳入现有对比度测试，文字与其所在纸面达到 WCAG AA。

### 4.2 圆角与阴影

- 一级分区统一使用现有 `--radius-md`，不引入大圆角视觉语言。
- 内层无框项目使用 `--radius-sm`。
- `--shadow-soft` 仅用于精选文章卡片和搜索弹窗。
- 吸顶页头使用比 `--shadow-soft` 更弱的 `--shadow-header`，浅色不透明度不高于 4%，深色不高于 10%。
- 普通首页区块、目录项目、文章列表项和文章尾部导航不得使用常驻阴影。

### 4.3 动效

保留现有时长、缓动、品牌动效和 `prefers-reduced-motion` 行为。无框列表项悬停只使用背景色、文字色和不超过现有幅度的位移；不得新增大面积缩放、发光或模糊。

## 5. 全站骨架

### 5.1 页头与页脚

- `.site-header` 移除底边线，通过半透明纸面、背景模糊和极轻阴影维持吸顶层级。
- 当前导航继续使用下划线，不能退化为纯颜色状态。
- `.site-footer` 移除顶边线，通过更清晰的上方留白与现有品牌排版完成页面收束。
- 页头、页脚的按钮、导航热区和移动端横向导航行为保持不变。

### 5.2 通用页面标题

- `.page-header` 移除底边线，改为 `--paper-soft` 背景、现有圆角和响应式内边距。
- 标题、eyebrow、说明文字的字号、颜色层级保持现有体系。
- 分类、专栏、标签详情页的返回链接继续位于标题区前方，不改变语义和顺序。

## 6. 首页与文章索引

### 6.1 首页一级分区

- `.home-hero`、`.home-taxonomy` 和每个 `.home-section` 移除底边线，分别成为独立的 `--paper-soft` 一级分区。
- 分区之间使用一致的外部间距；内部标题不再绘制 `.section-heading` 底线。
- 首页品牌 SVG、字体动效和真实内容顺序保持不变。

### 6.2 分类与专栏索引

- `.home-taxonomy__list` 移除顶线与逐项底线。
- 项目用网格对齐、行距和数量文字建立扫描秩序；悬停时使用 `--paper-interactive`。
- 分类与专栏仍保持两栏桌面、单栏手机布局。

### 6.3 精选卡片

- `.post-card` 移除外框，保留 `--paper-raised` 背景、现有圆角和一层轻阴影。
- `.post-card__cover` 移除封面与正文之间的分隔线。
- 卡片焦点状态继续显示清晰的外部焦点反馈；不得因移除常驻外框而丢失键盘位置。

### 6.4 文章行

- `.post-row-list` 移除列表顶线，改用小间距组织项目。
- `.post-row` 移除逐项底线和悬停左侧强调线。
- 项目默认透明，悬停或 `:focus-within` 时使用 `--paper-interactive`，并保留现有轻位移。
- 封面尺寸、摘要截断、延伸点击区域和分类链接层级不变。

## 7. 分类、专栏、标签与关于页

- `.taxonomy-directory__link` 与 `.tag-directory__link` 移除完整描边，使用 `--paper-soft`；悬停时转为 `--paper-interactive`。
- 普通 `.taxonomy-pill` 移除线框，改用低对比底色。文章分类继续使用砖红实底，专栏与标签通过文字和底色区分。
- 胶囊可见高度继续为 36px，透明扩展命中区域继续维持约 44px。
- `.about-page__grid section` 移除顶线，依靠编号、标题和间距区分。
- `.about-page__principle` 移除外框，保留 `--paper-soft` 与内边距，不使用常驻阴影。

## 8. 文章阅读页

### 8.1 文章头部与目录

- `.post-header` 移除底线，使用 `--paper-soft`、现有圆角与响应式内边距形成标题纸面。
- 文章正文继续直接位于 `--paper` 上，不包进卡片。
- 桌面 `.post-toc` 移除顶线，改为 `--paper-soft` 小型纸面区。
- 移动端 `.post-toc-compact` 移除上下双线，使用同一纸面区；原生 `<details>`、目录链接和 44px 热区不变。

### 8.2 文章尾部导航

- `.post-pagination`、`.post-series-navigation` 和 `.related-post-list` 移除区块顶线。
- 上一篇/下一篇和专栏相邻文章链接移除描边，使用 `--paper-soft`，悬停时使用 `--paper-interactive`。
- `.related-post` 移除逐项底线，以项目间距和悬停底色区分。
- 区域标题、进度、上一篇/下一篇语义和链接顺序不变。

### 8.3 正文基础元素

- 行内 `code` 与代码块 `pre` 移除外框，继续使用等宽字体、背景色和圆角。
- 表格行分隔、`blockquote` 左边线、作者主动插入的 `hr` 保留。
- 正文链接的颜色、下划线、焦点与换行行为不变。

## 9. 搜索与飞书内容边界

### 9.1 搜索

- 搜索弹窗外框、输入框边界、键盘焦点和当前结果轮廓保留。
- `.search-dialog__header` 底线与 `.search-dialog__result` 逐项顶线移除。
- 普通、悬停、键盘当前结果继续以背景色和轮廓形成清晰差异。
- 搜索脚本、快捷键、加载、空结果、错误和归档回退行为不变。

### 9.2 飞书富内容

以下边界属于内容语义，不参与全局减线：

- 飞书 Callout 的背景、边框颜色和文字颜色。
- Source Synced 块保留现有单层边界，背景可以适配新纸面 token，但不得无边界化或叠加阴影。
- 飞书文本下划线、链接下划线和颜色标注。
- 表格行分隔、引用线和作者插入的分隔线。

`src/styles/feishu-content.css` 只在需要适配新纸面 token 时调整，不得删除或弱化上述语义映射。

## 10. 响应式、主题与可访问性

- `48rem` 以下减少一级纸面的内边距，但不移除圆角或改变内容顺序。
- 手机端所有页面不得出现新增横向溢出；现有导航、表格和公式的受控横向滚动保持不变。
- 明暗主题必须呈现相同的层级关系；深色纸面不能与页面背景融为一体，也不能形成高亮色块墙。
- 全局 `:focus-visible` 3px 焦点环保持不变。
- 所有操作控件继续至少 44px；36px taxonomy 胶囊继续使用透明扩展命中区域。
- 当前导航、选中搜索结果等状态不得只依赖颜色。
- 减少动态模式继续直接显示稳定终态。

## 11. 实现结构与故障边界

实现以现有 class 和样式层为主：

- `src/styles/global.css` 负责新 token、全站分区、卡片、列表、目录、文章页和搜索样式。
- `src/styles/feishu-content.css` 仅在新纸面 token 影响富内容对比度时做最小适配。
- Astro 页面和组件默认不改 DOM；只有现有选择器无法表达经确认层级时，才允许增加纯表现 class，不能改变语义元素或数据流。
- 内容集合、路由、搜索索引、飞书转换、主题脚本和组件属性不变。

若新增 token 在某一主题未定义，测试必须失败；不得回退到透明背景或静默继承。若移除边线导致焦点、当前态或语义边界不可辨认，应恢复该具体边界，而不是整体恢复 `--line`。

## 12. 测试与验收

### 12.1 测试驱动契约

先在 `tests/ui-source.test.mjs` 和必要的 `tests/design-contrast.test.mjs` 中写入失败测试，再修改生产样式。测试至少覆盖：

- 两个主题均定义新纸面 token。
- 已列出的装饰性边界选择器不再声明常驻边线。
- 一级分区、无框列表项和交互状态使用正确纸面 token。
- 焦点环、当前导航下划线、搜索必要边界、表格、引用和飞书语义边界继续存在。
- 36px 胶囊可见高度和约 44px 命中区域不变。
- 响应式规则不重新引入已删除边线。

### 12.2 自动化门禁

- 运行相关源码与对比度测试。
- 运行完整 `npm run verify`。
- Astro 检查达到 0 errors、0 warnings、0 hints，生产构建成功。

### 12.3 浏览器验收

至少检查首页、文章归档、分类/专栏/标签目录、关于页、普通文章页和飞书富内容文章页：

- 桌面 1440px 与手机 320px。
- 浅色与深色主题。
- 页面无新增横向溢出，文本无裁切。
- 一级纸面层级清楚，组内没有重复线条或过多阴影。
- 悬停、键盘焦点、当前导航、搜索当前结果均可辨认。
- 表格、引用、hr、Callout、Source Synced 与公式保持准确。
- 浏览器控制台无新增错误或警告。

## 13. 非目标

- 不改变博客导航、路由、内容模型或页面信息顺序。
- 不修改飞书同步、富内容识别、搜索排序或文章发现逻辑。
- 不更换字体、品牌色、图标或动效体系。
- 不引入新依赖、外部字体、动画库或客户端 UI 框架。
- 不重新设计社交分享卡片。
- 不发布或推送线上站点；集成与部署继续使用现有分支流程。
