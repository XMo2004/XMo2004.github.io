# 分类、专栏与高密度编辑式改版设计

日期：2026-07-12
状态：用户授权自主设计并直接实施

## 1. 目标

在不改变 Astro 静态托管和飞书发布链路的前提下，为博客增加分类与专栏，并完成全站信息密度、品牌动效和文章阅读体验改版。

本轮同时落实此前收集的界面反馈：

- 首页标题改为“xmo 的博客”，使用克制的新式文字动效。
- 首页删除介绍段、两个操作按钮和旧“博客说明”卡片。
- 右侧改为镂空 XMO 品牌标志动效。
- 主题切换改为无方框的日相/月相图标。
- 页脚加入镂空品牌图标与进入视口时的文字动效。
- 桌面端文章标题尽量保持一行，文章头部明显变紧凑。
- 标签改为胶囊样式。
- 单章节不显示目录；多章节目录在桌面端位于正文右侧，移动端折叠。
- 首页、归档、标签、关于和文章页减少无效留白，提高扫描效率，但不压缩正文基本字号和触控面积。

## 2. 内容模型

### 2.1 分类

分类是文章唯一的主归属，每篇文章必须有且只有一个分类。分类适合表达“技术”“成长”“随笔”等稳定的大主题，不替代标签。

字段：

- Markdown frontmatter：`category: 技术`
- 飞书多维表格：`分类`，单选，发布时必填
- 公开路由：`/categories/` 与 `/categories/<slug>/`

分类聚合页按最新发布日期排序文章；目录页显示分类名、文章数量和最近更新时间。

### 2.2 专栏

专栏是可选的、有阅读顺序的文章系列。一篇文章最多属于一个专栏。

字段：

- Markdown frontmatter：`column: 博客搭建手记`
- Markdown frontmatter：`columnOrder: 1`
- 飞书多维表格：`专栏`（单选）与 `专栏序号`（正整数）
- 公开路由：`/columns/` 与 `/columns/<slug>/`

规则：`专栏` 与 `专栏序号` 必须同时存在或同时缺失；同一专栏内序号不能重复。专栏页按序号升序展示文章，序号相同时构建失败，不静默猜测顺序。

### 2.3 标签

标签继续作为可多选的细粒度关键词。分类、专栏、标签三者在语义和路由上保持独立：

- 分类回答“这篇文章主要属于什么主题”。
- 专栏回答“它是否属于一个连续系列，以及读到第几篇”。
- 标签回答“文章还涉及哪些具体概念”。

## 3. 初始内容迁移

- “欢迎来到小陌的博客”：分类为“随笔”，专栏为“博客搭建手记”，序号 1。
- “从飞书发布的第一篇文章”：分类为“技术”，专栏为“博客搭建手记”，序号 2。

飞书表新增三个字段后，先写入现有已发布记录，再部署要求新字段的同步代码，避免定时同步出现迁移窗口。

## 4. 信息架构

主导航调整为：首页、文章、分类、专栏、标签、关于。移动端导航允许横向滚动，不压缩文字或触控目标。

首页使用“编辑部索引”结构：

1. 紧凑品牌首屏：短标题与 XMO 镂空标志。
2. 分类与专栏索引条：直接展示真实数量。
3. 精选文章：保留封面，但压缩高度。
4. 最新文章：使用紧凑横向文章行。

文章归档和分类/标签详情统一使用横向文章行：日期、缩略图、标题、一行摘要、分类/专栏/标签与阅读信息。没有封面时不生成大面积年份占位图。

专栏目录展示专栏名、篇数和最近更新；专栏详情按序号展示，并明确标记“01 / 02”。

## 5. 视觉与动效

延续现有暖纸白、墨黑、陶土橙、鼠尾草绿与中文编辑式排版，不引入渐变、外部字体或动画库。

### 5.1 品牌动效

- “xmo”的字母像活字一样从轻微错位落入基线，“的博客”从陶土色裁切线后揭示。
- 镂空 XMO 标志由三组线性笔画组合，首次进入时完成一次描边与归位；悬停只做轻微重组。
- 动效只使用 `transform`、`opacity`、`clip-path` 和 SVG 描边，不使用大面积模糊或粒子。
- `prefers-reduced-motion: reduce` 下直接显示清晰终态。

### 5.2 密度原则

- 桌面端全局垂直留白减少约 25%–35%。
- 首页首屏从约 31rem 压缩至约 18–22rem，并让首屏下缘看到内容入口。
- 页面标题区、栏目标题和卡片间距统一压缩。
- 文章归档改为分隔线式信息行，减少厚卡片、阴影与空白占位。
- 正文宽度保持约 44–46rem，字号不低于 1rem，行高保持约 1.7–1.8。
- 所有交互目标继续保持至少 44px。

## 6. 文章阅读页

文章头部最大宽度扩大，桌面标题取消过窄的 `ch` 限制并使用响应式字号；较短标题优先单行，长标题仍可自然换行。

摘要、日期、阅读时间、分类、专栏和标签组合成紧凑信息区。分类使用实底小胶囊，专栏使用线框胶囊，标签使用柔和胶囊。

目录只统计有效的二至四级标题：

- 少于两个目录项：不显示目录，正文居中。
- 两个及以上：桌面端放在正文右侧并轻量吸顶。
- 64rem 以下：折叠为正文前的原生 `<details>`。

## 7. 数据与安全

飞书同步仅新增公开分类与专栏字段，不把飞书记录 ID、文档 ID 或内部 URL写入公开 frontmatter。分类和专栏名称使用现有稳定的 Unicode 路由规范化与冲突检查；异常或重复序号使整次同步失败并保留上一版站点。

分类/专栏页面完全由内容集合静态生成，不增加服务器、数据库或客户端请求。

## 8. 测试与验收

- 单元测试：分类/专栏聚合、Unicode slug、冲突、专栏排序和重复序号。
- 飞书测试：字段解析、必填关系、正整数序号、frontmatter 输出。
- 源码测试：导航、首页精简、品牌动效、主题按钮、胶囊标签、紧凑头部和目录阈值。
- 构建测试：分类/专栏目录与详情均生成，sitemap 收录新路由。
- 浏览器测试：桌面、平板、手机；浅色、深色、减少动态；标题不溢出、导航可用、控制台无错误。
- 线上测试：首页、新聚合页、文章页和静态资源均返回 200。
