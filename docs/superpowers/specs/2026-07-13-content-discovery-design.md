# 小陌博客：内容发现与连续阅读设计

日期：2026-07-13  
状态：已确认，待实施

## 1. 背景

博客已具备飞书发布、文章归档、分类、专栏、标签、静态部署与响应式阅读体验，但内容发现仍依赖逐页浏览。文章数量增加后，读者难以快速定位旧内容；文章页现有的全站上一篇/下一篇也无法表达专栏顺序，相关文章之间缺少稳定关联。

本阶段补齐三个互相协作的能力：站内搜索、专栏连续阅读、相关文章推荐。所有能力继续保持纯静态托管，不引入外部搜索服务、账号、密钥、追踪脚本或服务端运行时。

## 2. 目标与非目标

### 目标

- 读者可从任意页面用按钮或键盘快捷键打开搜索。
- 中文、英文与数字关键词均可检索公开文章的标题、摘要、分类、专栏、标签和正文。
- 专栏文章显示明确的阅读位置以及上一节、下一节。
- 非专栏文章保留普通时间相邻导航。
- 每篇文章可展示最多三篇不重复的相关推荐。
- 搜索、排序、推荐和专栏导航均可测试、确定性构建且不泄露飞书内部元数据。
- 桌面与移动端保持当前高密度编辑排版、44px 触控目标、深色模式和减少动态偏好。

### 非目标

- 不提供模糊拼写纠错、语义向量搜索或云端搜索。
- 不引入 Pagefind、Algolia、Meilisearch 或前端框架。
- 不增加登录、收藏、阅读历史、评论或个性化推荐。
- 不改变飞书 Base 字段或当前文章发布工作流。
- 不在本阶段引入访问统计或行为追踪。

## 3. 方案选择

### 采用：自建静态 JSON 索引

Astro 构建时生成同源 `/search-index.json`，浏览器仅在首次打开搜索时加载。搜索由少量原生 JavaScript 完成，使用明确的字段权重和稳定排序。

选择理由：

- 与纯静态托管和隐私边界一致。
- 当前内容规模不需要第三方索引器。
- 中文子串匹配无需额外分词服务。
- 索引结构、排序规则、失败回退和公开字段完全可控。

### 未采用：Pagefind

Pagefind 提供成熟的离线全文搜索，但会新增依赖、生成资产和另一层索引约定。当前站点规模下收益不足以抵消复杂度。

### 未采用：托管搜索服务

Algolia 等服务适合大型内容库，但需要外部账号、密钥、网络依赖及潜在追踪，与当前博客的简单部署目标不符。

## 4. 静态搜索索引

### 路由与数据契约

新增 `/search-index.json` 静态端点，响应 `application/json; charset=utf-8`。顶层结构固定为：

```json
{
  "version": 1,
  "entries": []
}
```

每个条目只包含公开字段：

- `href`
- `title`
- `description`
- `pubDate`
- `category`
- 可选 `column` 与 `columnOrder`
- `tags`
- `readingMinutes`
- `searchText`

索引不得包含飞书 record ID、document ID、file token、原文内部 URL、manifest 数据或本地文件路径。条目按发布日期降序、公开 slug 升序稳定排列，不写入构建时间等非确定性字段。

### 正文转换与体积

构建时把公开 Markdown 正文转换为适合检索的纯文本：

- NFKC 规范化。
- 去除 Markdown 控制符、链接目标和 HTML 标签，保留可见文字与代码内容。
- 合并重复空白。
- 每篇正文进入 `searchText` 的部分最多 12,000 个字符；标题、摘要和分类元数据不截断。

索引只在搜索首次打开时请求，并在当前页面会话内复用。搜索功能不阻塞首屏渲染。

## 5. 搜索匹配与排序

### 查询规范化

- 查询先做 NFKC、转小写、首尾去空白并合并连续空白。
- 中文可直接使用连续子串匹配。
- 含空格的查询拆为多个词项；每个词项必须至少命中一个公开字段。
- 空查询不执行搜索，而展示最近更新文章。
- 前端最多显示八条结果。

### 权重

采用确定性字段权重：

1. 标题完全匹配或前缀匹配。
2. 标题包含。
3. 分类、专栏或标签匹配。
4. 摘要匹配。
5. 正文匹配。

同分时按发布日期降序，再按 `href` 升序。实现必须是无副作用的纯函数，不修改索引输入。

搜索结果展示标题、摘要、日期、分类、专栏和标签；不尝试生成正文命中高亮片段，以控制客户端复杂度和视觉噪声。

## 6. 搜索交互

### 入口

页头在主题按钮旁新增一个与现有镂空视觉一致的搜索按钮。按钮拥有可见焦点和 `aria-label="搜索文章"`，桌面可显示快捷键提示，移动端只显示图标。

支持：

- `Command + K`
- `Control + K`
- `/`

当焦点位于输入框、文本域、选择器或可编辑区域时，`/` 不拦截原输入。

### 对话框

使用原生 `<dialog>` 构建搜索层：

- 打开后焦点进入搜索框。
- `ArrowUp`、`ArrowDown` 移动当前结果。
- `Enter` 打开当前结果。
- `Escape` 关闭并把焦点还给触发按钮。
- 点击遮罩可关闭。
- 结果变化通过克制的 `aria-live` 文本报告数量。

视觉采用紧凑的编辑索引面板，不使用大面积模糊玻璃或重阴影。打开和结果进入只使用 opacity 与 transform；`prefers-reduced-motion: reduce` 下直接显示最终状态。

### 渐进增强与失败回退

- JavaScript 尚未准备好时不显示不可用按钮。
- 索引加载失败时显示“搜索暂不可用”，并提供文章归档链接。
- 无结果时展示清晰空状态，不伪造推荐结果。
- 搜索失败不影响导航、文章正文和其他静态页面。

## 7. 专栏连续阅读

新增纯函数为专栏文章计算：

- 专栏名称与专栏归档链接。
- 当前位置和文章总数，例如 `02 / 05`。
- 上一节：同专栏中更小的 `columnOrder`。
- 下一节：同专栏中更大的 `columnOrder`。

专栏文章底部使用专栏导航替代现有的时间相邻导航，文案为“上一节 / 下一节”。导航同时提供返回专栏目录的入口，并保留 `rel="prev"` / `rel="next"`。

非专栏文章继续使用现有“上一篇 / 下一篇”。若专栏只有一篇，仍显示 `01 / 01` 与专栏归档入口，但不渲染空链接。

## 8. 相关文章

新增纯函数为当前文章生成最多三篇相关推荐。候选先排除：

- 当前文章。
- 已在页面专栏导航或普通相邻导航中出现的文章。

确定性评分顺序：

1. 同专栏。
2. 同分类。
3. 每个共同标签。
4. 发布日期接近度仅用于辅助排序。

同分时按发布日期降序，再按公开 `href` 升序。分数为零的候选不展示；没有合格候选时整段省略。推荐结果复用紧凑文章行的视觉语言，但使用独立的“继续阅读”区域，不与正文或分页混排。

## 9. 组件与职责

- `src/pages/search-index.json.ts`：从公开文章集合生成确定性 JSON。
- `src/lib/search.ts`：Markdown 文本化、查询规范化、匹配与评分纯函数。
- `src/components/SearchDialog.astro`：对话框结构、加载状态、键盘交互与结果渲染。
- `src/components/SearchToggle.astro`：页头搜索入口。
- `src/lib/posts.ts`：新增专栏导航和相关文章的纯计算 helper。
- `src/pages/posts/[...id].astro`：在构建路由时计算文章级发现数据。
- `src/layouts/PostLayout.astro`：渲染专栏导航、普通相邻导航和继续阅读。
- `src/styles/global.css`：复用现有 token 添加搜索及文章尾部样式。

搜索模块与文章推荐模块互不依赖；它们只共享现有的公开文章字段和 URL helper。

## 10. 安全、隐私与错误处理

- 浏览器只请求本站同源静态 JSON。
- 不发送查询词，不写 cookie，不持久化搜索历史。
- 索引生成前继续依赖内容 schema、route collision 和 taxonomy 校验。
- JSON 序列化必须安全处理 `<`、`>`、`&` 等字符，避免内容被解释为脚本或 HTML。
- 公开索引测试必须扫描现有飞书内部标识模式和可信飞书内部 URL。
- 任一文章缺少公开正文时构建失败，不静默生成残缺索引。

## 11. 可访问性与响应式要求

- 搜索入口、关闭按钮和结果链接的触控尺寸不小于 44px。
- 对话框具备明确标题、描述、状态文本和焦点顺序。
- 键盘可完成打开、输入、选择、进入和关闭全流程。
- 320px 宽度下不得产生页面级横向滚动。
- 搜索结果元数据允许换行，标题不做导致内容不可见的强制单行截断。
- 深色模式保持现有纸张层级和 AA 对比度。

## 12. 验收标准

- 搜索“飞书”能找到《从飞书发布的第一篇文章》。
- 搜索“博客搭建”能通过专栏字段找到两篇现有文章。
- `Command/Ctrl + K`、`/`、方向键、Enter 和 Escape 均按设计工作。
- 首次打开搜索才请求索引；再次打开不重复请求。
- 当前第一篇飞书文章显示专栏位置 `02 / 02`，上一节链接到欢迎文章，无空的下一节。
- 欢迎文章显示 `01 / 02`，下一节链接到飞书文章。
- 页面中相邻导航与相关文章不出现重复链接。
- 空推荐段不渲染占位框。
- 搜索索引、sitemap 和所有新增页面构建成功，公开产物不含飞书内部元数据。
- 单元测试、Astro check、静态构建、桌面与移动浏览器验收、线上 HTTP 验收全部通过。
